76  * LZ4_MEMORY_USAGE :
77  * Memory usage formula : N->2^N Bytes (examples : 10 -> 1KB; 12 -> 4KB ; 16 -> 64KB; 20 -> 1MB; etc.)
78  * Increasing memory usage improves compression ratio
79  * Reduced memory usage can improve speed, due to cache effect
80  * Default value is 14, for 16KB, which nicely fits into Intel x86 L1 cache

92 /*
93 LZ4_compress_default() :
94  Compresses 'sourceSize' bytes from buffer 'source'
95  into already allocated 'dest' buffer of size 'maxDestSize'.
96  Compression is guaranteed to succeed if 'maxDestSize' >= LZ4_compressBound(sourceSize).
97  It also runs faster, so it's a recommended setting.
98  If the function cannot compress 'source' into a more limited 'dest' budget,
99  compression stops *immediately*, and the function result is zero.
100  As a consequence, 'dest' content is not valid.
101  This function never writes outside 'dest' buffer, nor read outside 'source' buffer.
102  sourceSize : Max supported value is LZ4_MAX_INPUT_VALUE
103  maxDestSize : full or partial size of buffer 'dest' (which must be already allocated)
104  return : the number of bytes written into buffer 'dest' (necessarily <= maxOutputSize)
105  or 0 if compression fails
106 
107 LZ4_decompress_safe() :
108  compressedSize : is the precise full size of the compressed block.
109  maxDecompressedSize : is the size of destination buffer, which must be already allocated.
110  return : the number of bytes decompressed into destination buffer (necessarily <= maxDecompressedSize)
111  If destination buffer is not large enough, decoding will stop and output an error code (<0).
112  If the source stream is detected malformed, the function will stop decoding and return a negative result.
113  This function is protected against buffer overflow exploits, including malicious data packets.
114  It never writes outside output buffer, nor reads outside input buffer.
115 */

136 /*
137 LZ4_compress_fast() :
138  Same as LZ4_compress_default(), but allows to select an "acceleration" factor.
139  The larger the acceleration value, the faster the algorithm, but also the lesser the compression.
140  It's a trade-off. It can be fine tuned, with each successive value providing roughly +~3% to speed.
141  An acceleration value of "1" is the same as regular LZ4_compress_default()
142  Values <= 0 will be replaced by ACCELERATION_DEFAULT (see lz4.c), which is 1.
143 */

287 /*
288 *_continue() :
289  These decoding functions allow decompression of multiple blocks in "streaming" mode.
290  Previously decoded blocks *must* remain available at the memory position where they were decoded (up to 64 KB)
291  In the case of a ring buffers, decoding buffer must be either :
292  - Exactly same size as encoding buffer, with same update rule (block boundaries at same positions)
293  In which case, the decoding & encoding ring buffer can have any size, including very small ones ( < 64 KB).
294  - Larger than encoding buffer, by a minimum of maxBlockSize more bytes.
295  maxBlockSize is implementation dependent. It's the maximum size you intend to compress into a single block.
296  In which case, encoding and decoding buffers do not need to be synchronized,
297  and encoding ring buffer can have any size, including small ones ( < 64 KB).
298  - _At least_ 64 KB + 8 bytes + maxBlockSize.
299  In which case, encoding and decoding buffers do not need to be synchronized,
300  and encoding ring buffer can have any size, including larger than decoding buffer.
301  Whenever these conditions are not possible, save the last 64KB of decoded data into a safe buffer,
302  and indicate where it is saved using LZ4_setStreamDecode()
303 */
304 int LZ4_decompress_safe_continue (LZ4_streamDecode_t* LZ4_streamDecode, const char* source, char* dest, int compressedSize, int maxDecompressedSize);
